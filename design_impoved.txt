# MediCard 医杀（V5.2 最终开发版）

# 核心原则：双向答题对战、极简规则、纯医学驱动、三国杀精髓

================================================================================
## 一、核心基础规则（必须先实现）
================================================================================
1. 唯一资源：生命值(HP)，1HP=1颗❤️
2. 初始血量：主公5血，其他所有身份4血
3. 手牌上限 = 当前血量（最低为1）
4. 任何时候手牌数 > 当前血量 → 弃牌阶段必须弃至等于血量
5. 胜利条件：将对手/敌方血量打到0
6. 核心机制：
   - 攻击牌：你打出 → 对手答题 → 答错受伤害，答对闪避
   - 防御牌：被攻击时打出 → 你答题 → 答对免疫，答错失效
   - 其他所有牌：自己打出 → 自己答题 → 答对生效，答错作废
7. 答错惩罚：仅卡牌作废，不扣血、不反噬
8.特殊牌点击可查看用法

================================================================================
## 二、游戏模式
================================================================================
### 2.1 单人练习模式
- 玩家 vs AI
- 玩家先手，身份为主公，AI为反贼
- AI逻辑：随机打出可使用的牌，答题正确率随难度调整
- 难度选择：简单(AI正确率50%)/普通(70%)/困难(90%)

### 2.2 联机模式
- 支持2/3/4/5人身份局
- 房主创建房间，生成6位房间号
- 其他玩家输入房间号加入
- 人数满后房主开始游戏，随机分配身份

### 2.3 身份配置与胜利条件
| 人数 | 身份配置 | 胜利条件 |
|------|----------|----------|
| 2人 | 1主公 + 1反贼 | 主公：击杀反贼；反贼：击杀主公 |
| 3人 | 1主公 + 1忠臣 + 1内奸 | 主公+忠臣：击杀内奸；内奸：先杀忠臣，最后杀主公 |
| 4人 | 1主公 + 1忠臣 + 2反贼 | 主公+忠臣：击杀所有反贼和内奸；反贼：击杀主公；内奸：先杀所有人，最后杀主公 |
| 5人 | 1主公 + 1忠臣 + 2反贼 + 1内奸 | 同上 |

### 2.4 身份技能
- 主公👑：初始5血
- 忠臣🛡️：主公受到攻击时，可选择替主公挡刀（自己打出防御牌并答题）
- 反贼💀：攻击主公时伤害+1
- 内奸🕵️：每回合多摸1张牌

## 三、完整回合流程
================================================================================
1. 回合开始
2. 判定阶段
   - 检查是否有延时锦囊（病毒感染/麻醉剂/疫苗接种）
   - 对应玩家答题：答对=效果失效；答错=效果生效
3. 摸牌阶段
   - 从牌库顶摸2张牌
   - 手牌已满则跳过
4. 出牌阶段
   - 可出任意多张牌，顺序不限
   - 每次出牌触发对应答题流程
   - 攻击牌只能攻击距离1以内的玩家（装备可改变距离）
5. 响应阶段（攻击/锦囊打出时触发）
   - 被攻击玩家可选择打出防御牌
   - 响应顺序：从当前回合玩家开始，按座位顺时针依次响应
   - 响应时间限制：15秒
6. 弃牌阶段
   - 弃置手牌至等于当前血量
   - 系统自动提示需弃置的牌数
7. 回合结束

## 四、完整卡牌体系（共120张基础牌库）
================================================================================
### 4.1 卡牌通用属性
- cardId: 唯一ID
- cardName: 卡牌名称
- cardType: 基础/锦囊/装备/延时锦囊
- icon: 图标(emoji)
- effect: 效果描述
- answerPlayer: 答题方(self/opponent/all)
- rarity: 普通/稀有/史诗
- subject: 对应医学学科(细胞/生化/生理/病理/组胚/解剖/免疫/微生物)

## 五、答题系统规则
================================================================================
1. 题目类型：医学选择题（4选1）/判断题（2选1）
2. 答题时限：
   - 普通题：15秒
   - 稀有题：20秒
   - 史诗题：25秒
3. 题目难度与卡牌稀有度对应：
   - 普通卡牌：基础题
   - 稀有卡牌：进阶题
   - 史诗卡牌：难题
4. 答题流程：
   - 弹出答题弹窗，显示题目和选项
   - 倒计时开始
   - 玩家选择答案
   - 答对：绿色高亮正确选项，卡牌效果生效
   - 答错：红色高亮错误选项，绿色显示正确答案，卡牌作废
   - 显示题目解析和教材出处
   - 点击"继续"关闭弹窗
5. 时间耗尽未作答 = 视为答错

## 六、UI界面设计要求
================================================================================
### 6.1 通用要求
- 主题：暗色系赛博医学风
- 玻璃拟态UI面板
- 响应式设计，支持移动端和桌面端
- 触摸目标最小44px

### 6.2 标题画面
- 游戏名称：MediCard 医杀
- 三个按钮：单人练习 / 联机对战 / 学习统计
- 显示版本号、学科数、总题数
- 学习统计弹窗：总局数、胜场、胜率、近10场正确率、错题数

### 6.3 科目选择画面
- 8个医学学科卡片，带图标和题目数量
- 4个快捷预设：形态学包/功能学包/病原防御包/全部8科
- 实时统计：已选N科、总题数、预计游戏时长
- 确认按钮

### 6.4 联机大厅画面
- 输入昵称
- 创建房间按钮
- 加入房间按钮（展开6位数字输入框）
- 房间内玩家列表
- 准备按钮
- 开始游戏按钮（仅房主可见）

### 6.5 对战画面
- 上区：对手信息（名称、身份、❤️血量、装备区）
- 中区：战场（回合指示器、阶段文字、延时锦囊区）
- 下区：你的信息（名称、身份、❤️血量、装备区）
- 手牌区：扇形展开，超上限时红色高亮
- 操作按钮：结束回合
- 响应弹窗：被攻击时弹出，显示"防御"和"不防御"按钮

### 6.6 结算画面
- 胜/败展示（奖杯/骷髅图标）
- 本局统计：答题正确率、答对题目数、造成伤害数
- 按钮：再来一局 / 查看错题 / 返回标题

## 七、数据持久化
================================================================================
使用localStorage存储，所有key以"medicard_"前缀：
- medicard_selected_subjects: 上次选择的科目列表
- medicard_player_name: 玩家昵称
- medicard_game_stats: 对局统计数组，每条含{won, accuracy, subjects, date}
- medicard_wrong_questions: 答错的题目ID数组

## 八、联机架构（基于PeerJS P2P）
================================================================================
### 8.1 信令服务器
- 使用公共PeerJS服务器：0.peerjs.com:443
- PeerID格式：medicard-{6位房间号}-host / medicard-{房间号}-client-{随机6字符}

### 8.2 消息协议（JSON）
| 消息类型 | 方向 | 说明 |
|----------|------|------|
| JOIN_REQUEST | C→H | 客户端请求加入房间 |
| JOIN_ACCEPT | H→C | 房主接受，含玩家列表 |
| JOIN_REJECT | H→C | 房主拒绝（房间已满） |
| PLAYER_READY | C→H | 客户端准备就绪 |
| PLAY_CARD | C→H | 客户端报告出牌和答题结果 |
| RESPOND_CARD | C→H | 客户端报告响应和答题结果 |
| END_TURN | C→H | 客户端结束回合 |
| GAME_START | H→C | 游戏开始，含初始手牌和身份 |
| TURN_CHANGE | H→C | 回合切换，含抽牌和状态 |
| GAME_OVER | H→C | 游戏结束通知 |
| PING/PONG | 双向 | 心跳保活 |

## 十、注意事项
================================================================================
1. 所有卡牌效果必须通过答题触发，无任何免费操作
2. 答错仅卡牌作废，不扣血，保持学习友好
3. 界面简洁，只显示必要信息，避免杂乱
4. 确保移动端体验良好，支持横屏和竖屏
5. 所有题目从选中的学科题库中随机抽取
6. 牌库耗尽时自动洗入弃牌堆
